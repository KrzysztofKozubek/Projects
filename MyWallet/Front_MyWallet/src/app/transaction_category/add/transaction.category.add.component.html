Transaction categorization aids the retail user to categorize or split (more than one category) a particular transaction, which will be used for expense and budget analysis. Categorization or split of transaction will be available in each account type (Account summary, operative accounts, deposit accounts, loan accounts, credit cards and other bank accounts). In Transaction Categorization, retail user is enabled to categorize his expenses and income under defined categories which the user has linked to his account. This helps retail user to keep track of their spending and income. This screen allows the retail user to categorize or split the transactions that were initiated for the selected accounts. The transaction list by default would be shown for last three months transaction for the account chosen. However, this is a configurable parameter and retail user can list the transactions based on the search criteria also. The result list shows the bank name, account number, date of transaction, remarks, category, amount type, amount and whether the transaction is split or not. Retail user can choose the transactions and update the category of the transactions according to his need. Any transaction which is not categorized would by default remain as Uncategorized. Retail user will also not be able to set any budget details for this category. Retail user can choose any records and update the category or can split any transactions. For already split transactions, the category column is disabled and is displayed as Uncategorized. However, retail users can update the already split transactions using Split Transaction function.
